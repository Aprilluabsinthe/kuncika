Javascript Turtle Graphics

Language Reference

  • forward(distance) | fd(distance)

    Move the turtle forward distance pixels.

    Example: forward (10)

  • backward(distance) | back(distance) | bk(distance)

    Move the turtle backward distance pixels, but leave it pointing forward.

    Example: backward (10)

  • right(angle) | rt(angle) | turn(angle)

    Turn the turtle to the right angle degrees.

    Example: right (90)

  • left(angle)

    Turn the turtle to the left angle degrees.

    Example: left (90)

  • penup() | up() | pu()

    Lift the pen in the turtle so that it does not leave a mark as it moves.

    Example: penup()

  • pendown() | down() | pd()

    Lower the pen in the turtle so that it leaves a mark as it moves.

    Example: pendown()

  • hideturtle() | ht()

    Hide the turtle. It can still draw, but it isn't shown.

    Example: hideTurtle()

  • showturtle() | st()

    Show the turtle.

    Example: showTurtle()

  • home()

    Move the turtle to its home in the center (x=0, y=0) and pointing toward the top (angle=0).

    Example: home()

  • clear()

    Clear the graphics area, but leave the turtle where it is.

    Example: clear()

  • reset()

    Clear the graphics area and move the turtle to its home in the center (x=0, y=0) and pointing toward the top (angle=0).

    Example: reset()

  • goto(x,y) | setposition(x,y) | setpos(x,y)

    Move the turtle to the coordinates x pixels to the right of the center of the drawing area (or -x pixels to the left of the center) and y pixels above the center of the drawing area (or -y to below the center). This does not draw on the way.

    Example: goto (20,10)

  • setx(x)

    Move the turtle horizontally to the coordinates x pixels to the right of the center of the drawing area (or -x pixels to the left of the center).

    Example: setx (0)

  • sety(y)

    Move the turtle vertically to the coordinates y pixels above the center of the drawing area (or -y to below the center). This does not draw on the way.

    Example: sety (0)

  • angle(angle) | setheading(angle) | seth(angle)

    Turn the turtle to to the absolute angle relative to top of the graphic area. 0 is the top, 0 is toward the top, 90 is toward the right, 180 is toward the bottom, and 270 -90 is toward the left.

    Example: angle (90)

  • width(width) | pensize(width) | penwidth(width) |

    Change the width of the turtle pen (in pixels).

    Example: width(5);

  • color(colorcode)

    Change the color of the turtle pen. The colorcode may be a number from 0 to 15 representing the standard turtle graphics colors.

    The colorcode may also be a quoted string with any acceptable CSS color code as in:

    • "colorName" where colorName is any of the named CSS colors.
    • "#RRGGBB" where RR, GG, and BB is the hexidecimal amount of red, green and blue respectively.
    • "rgb(RR, GG, BB)" where RR, GG, and BB is the decimal amount of red, green and blue respectively.
    • "rgba(RR, GG, BB, AA)" where RR, GG, and BB is the decimal amount of red, green and blue respectively and AA is the alpha channel for transparancy between 1.0 for opaque and 0.0 for totally transparent.
    • "rbg(RR, BB, GG)"
    • "rgba(RR, GG, BB, AA)"
    • "hsl(120, 100%, 50%)" where the values are hue is postion on the color wheel (0 to 360), saturation percentage, and lightness percentage.
    • "hsl(120, 100%, 50%, 1)" where the values are hue is postion on the color wheel (0 to 360), saturation percentage, lightness percentage and alpha channel for transparancy between 1.0 for opaque and 0.0 for totally transparent.

    Example: color("red")

  • write(string)

    Places the text in the stringalong the projected path of the turtle. The string may be a set of characters enclosed by single or double quotation marks. The string may also be a value returned by a function.

    Example: write("Hello World")

  • setfont(fontName)

    Set the characteristics (style, variant, weight, size, and font-family desired for the font for subsequent writes.

    Example: setfont("italic small-caps bold 12px courier")

  • curveright (radius[,extent])

    Draw a circular curve from the turtle's postion. The center of the curve is to the right of the turtle at a perpendicular distance of radius pixels. The curve is a full circle unless extent is provided to limit the number of degrees of arc.

    Example: curveright(30)

    Example: curveright(10,45)

  • curveleft (radius[,extent])

    Draw a circular curve from the turtle's postion. The center of the curve is to the left of the turtle at a perpendicular distance of radius pixels. The curve is a full circle unless extent is provided to limit the number of degrees of arc.

    Example: curveleft(20)

    Example: curveleft(10,45)

  • circle (radius[[,(extent],(CCW])

    Draw a circle or arc centered on the turtle's postion. A full circle is drawn unless extent is provided to limit the number of degrees of arc. The arc is drawn clockwise unless a boolean CCW is supplied and has a value of false.

    Example: circle(10)

    Example: circle(20,90)

    Example: circle(30,-90,false)

  • dot ([radius])

    // radius in pixels (if none, max of pensize+4, 2*pensize) Draw a filled circle centered on the turtle's postion. The circle is drawn with a radius of radius) pixels. If radius) is not specified, it defaults to the larger of width plus 4 or twice the pensize).

    Example: dot()

    Example: dot(10)

  • n = random([low,]high)

    Pick a random number between low and high -1. If only one value high is provided, pick a number between 0 and high -1.

    Example: write(random(1,7)+ " ")

    Example: write(random(6)+ " ")

  • repeat(n, action)

    Allows a function action or group of actions to be repeated n times.

    Example: repeat (4, function () {forward(10);right(90)})

    Example: function el () {forward(10);right(90)}; repeat (4, el)

  • wrap(bool)

    Turns wrapping on and off at the edges of the graphic area by setting it true or false respecitively.

    Example: wrap(true)

    Example: wrap(false)

  • animate(action,ms)

    Evalutes action every ms milliseconds.

    Example: animate ("forward(10)",100)

    Example: animate ("forward(100);right(90);",100)

    Example: function el () {forward(100);right(90)}; animate (el,100)

  • delay(action,ms)

    Evalutes action after a delay of ms milliseconds.

    Example: delay ("forward(10)",1000)

    Example: delay ("forward(100);right(90);",1000)

    Example: function el () {forward(100);right(90)}; delay (el,100)

  • redrawOnMove(bool)

    Turn redrawing on or off by specifying the bool as true or false.

    Example: redrawOnMove(true)

    Example: redrawOnMove(false)

  • draw()

    Draw the turtle and the current image.

    Example: draw()

  • n = minX()

    Return the minimum value of x in pixels

    Example: write(minX())

  • n = maxX()

    Return the maximum value of x in pixels

    Example: write(maxX())

  • n = minY()

    Return the minimum value of Y in pixels

    Example: write(minY())

  • n = maxY()

    Return the maximum value of Y in pixels

    Example: write(maxY())

The language reference lists the available functions for turtle graphics. Parameters are included within parentheses '(' and ')' characters. Optional parameters and separating commas are enclosed withing square bracket '[' and ']' characters. Alternative forms of the functions are separated with the pipe '|' character.

Canvas

The canvas is the area where the turtle graphics are drawn. The turtle is the green triangle.

The Command area is a place where text can be entered and executed by pressing the ENTER (Return) key.


The Reset button is used to return the canvas and the turtle to their starting positions.

The Run Demo button is used to run the function in the definitions area named demo().

The Stop button is used to stop animations. The Stop button only appears when it is needed.

The examples select is used to load various examples into the defintions area. The example can be run with the run demo button. Individual functions can be run by invoking them from the command area. The code can be examined or modified as desired.

The definitions area allow the entry of multi-line programs and examples. This code can be either as individual functions by invoking the function with the command area. Most example code includes a function named demo that can be invoked without parameters with the Run Demo button.

Basic Javascript Grammar

While a better and more complete guide to the Javascript grammar can be found elsewhere in books or on the Internet, this is a brief guide to help get you started.

Statements

Usually there is one Javascript statement per line, but you can put more on a line as long as each statement is separated with a semicolon ';'. Some programmers like to end each statement with a semi-colon anyway. Some statements may span more than one line, especially complex assignment statements. a = 1; a = 1 // statement without the ending semicolon a = 1; b = 2 a = 1 // could be the end or not depending on what follows + 1 // appended to preceding line + 1 // appended to the preceding lines + 2; // appended to the preceding lines, so a = 5

Many of Javascript's constructs come in pairs. Strings are enclosed in pair of single quotes (apostrophes) or a pair of double quotes. A single quote may appear in a string set off with double quotes and a double quote may appear in a string set of with single quotes. An opening parentheses, square bracket, or squiggle bracket (brace), must be matched with the closing parentheses, square bracket or squiggle bracket.

Comments

A comment is a note left by the developer to others, including the developer, about some aspect of the code to make it easier to understand. Comments are not executed by the Javascript interpreter. Javascript has two kinds of comments. A single line comment starts with a double slash '//' and runs to the end of the current line. The line may contain other Javascript before the double slash. A multi-line comment is fenced in by a slash-star '/*' start sequence and ends in a '*/' sequence. This comment may cover several lines or it may be within the Javascript code of a single line.

/* a multi-line comment that spans more than one line */ //a simple single line comment a = 1; // a single line comment at the end of a line a /* an embedded multi-line comment */ = 1; /* multi-line comments /* cannot be nested */ syntax error */

Variables

Variables are used to store a value for later use. Sometimes this may be a fairly long time and sometimes it may be vary short time. Variables do not have a single particular type. So the same variable may be assigned a number, such as 1, in one instance and be assigned a string like "Javascript" at some other time. It is up to the programmer to keep track of the type of a variable.

a = 1; // assign the value / assign the value of the integer '1' to the variable 'a'. a = 1.0; // assign the value of the floating point number ' 1.0' to the variable 'a'. a = "red rutabaga"; // assign the string "red rutabaga" to the variable 'a'. a = true; // assign boolean value of true to the variable 'a'.

Javascript allows for a wide variety of types beyond simple integers and strings. It allows floating point numbers, Boolean (true or false) values, ordered lists, and objectnctions for later use. This allows the same code to be used over and over. Some functions may be return a value, others do ns. You can find out more about these in other places.

Functions

Javascript allows the programmer to define blocks of code that can be used over and over. These blocks are called functions. Some functions may take parameters as inputs, others do not. It is up to the programmer to decide what the function will do, whether it takes parameters and whether it returns a result. A function may be called a procedure, a subroutine, or other names. These names are generally talking about the same thing that Javascript calls a function. A function is "invoked" in a statement by using its name followed by a opening and closing parentheses '(' and ')'. There may be parameters between the parentheses.

A simple example could be a function that draws a shape (like a square or other figure) on the canvas. The turtle is moved to where the shape is required and the function is invoked.

function square () { forward (100); right (90); forward (100); right (90); forward (100); right (90); forward (100); right (90); }

Good functions have no side effects. So a function that draws a shape should leave the turtle in the same position as it was when the function was called and pointing in the same direction. Of course there are exceptions like a function that draws a fancy corner. The point is, the function should be predictable and not have a side effect which would manifest itself as a nasty bug down the road.

Functions with Parameters

A more powerful version of a function is a function that is called with one or more parameters. A parameter may be a size of a shape. It may be the input of a mathmatical function, say one that takes a number as its input and returns the value of the square of the input (the value of the input times itself). There is no limit to the number of paramters that a function can use, but it is usually wise to keep the number down to the minimum reguired for the function.

function square (size) { forward (size); right (90); forward (size); right (90); forward (size); right (90); forward (size); right (90); }

Basic Javascript Concepts

Scoping

Let's say you want to use the variable 'i' as within an iteration as is common practice. There could be trouble if there is a variable named 'i' elsewhere in the program. To solve this problem generally languages, like Javascript, provide rules of scoping to limit the reach of a particular variable. In general a variable may be used within the context in which it is defined. A variable declared with 'var' within a function is only accessed within that function, even if there is another variable named 'i' in the code invoking that function. The 'var' declaration is important, as it states, this variable is defined here and connot be accessed out side of this scope. A function defines a scope basically as the code between the opening and closing curley brackets '{' and '}'. A variable that is defined at the same level of the function is called a "global" variable, in that it CAN be accessed from any function that does not include a locally defined variable of the same name. When a variable is used in a function without being defined with the 'var', it is assumed to be a global external to the function.

var angle = 90; // this is global var size = 10; // this is global function square (size /* parameters are automatially local variables) { forward (size); // size references the local parameter size right (angle); // angle references the global variable angle forward (size); right (angle); forward (size); right (angle); forward (size); right (angle); } square (100); // draws a square of size 100 pixels

Of course functions may be defined within the scope of a function. The "inner" functions can only be used by other inner functions or the outer function. It cannot be used by the code external to the outer function. Likewise variables can be declared within functions to limit their scope of usage.

Iteration

Computer science calls the process of doing things over and over, iteration. Turtle graphics give you only one way to do iteration with the repeat(count,function) statement. This lets you do things a set number of times. But what if you want to do things a number of times bases on some other criteria, like getting too small or too big. Then you need to look into Javascript for such purposes.

function square (size) { repeat (4, function () { forward (size); right (90); }) }

function spiral (count, size) repeat (count, function () { forward (size); right (91); }) }

function spiral (count, size) { for (var i = 0; i < count; i++) { forward (i); right (90); } }
